  1. 抽象类和接口的区别：

答：

​ 1.接口只能包含抽象方法，而抽象类可以包含普通方法。

​ 2.接口只能定义静态常量，而抽象类即可以定义普通属性。也可以定义静态常量属性。

​ 3.接口不包含构造方法，抽象类可以包含构造方法。

  1. hashcode和equals的重要性：

答：

​ 1. 同一个对象无论何时调用hashcode(),得到的返回值必须一样。

​ 2.hashcode()返回值相等，对象不一定相同。通过hashcode()和equals()必须唯一确定一个对象。

​ 3.一旦重写equals，必须重写hashcode.

6.三种排序

答：

  1. 冒泡排序

for（int i=0;i<length-1;i++）{

for(int j=0;j<length-1-i;j++){

if(array[j]>array[j+1]){

int temp=array[j];

array[j]=array[j+1];

array[j+1]=temp;

}

}

}

  1. 选择排序

for(int i=0;i<length-1;i++){

int minIndex=i;

for(int j=i;j<length;j++){

if(array[minIndex]>array[j])

minIndex=j;

}

int temp=array[i];

array[i]=array[minIndex];

array[minIndex]=temp;

}

  1. 插入排序(是简单快排中最快的)

for(int i=1;i<length;i++){

int temp=array[i];

int leftIndex=i-1;

while(leftIndex>=0&&array[leftIndex]>temp){

array[leftIndex+1]=array[leftIndex];

leftIndex- -;

}

array[leftIndex+1]=temp;

}

金证财富面试经验

今天怀着平常心去了真正意义上的第一次助理java工程师面试，我很清楚自己技术上有特别多的不足，所以我只是纯粹想体验一下被面试的感觉。虽然早就知道自己肯定不会入选，但被无情地淘汰的确有点心疼。嗯，从中吸取点经验教训吧，这才是不枉自己走的这一趟。

  1. 好像技术面试官问了我关于协议方面的知识，我并没有回答出来。

  2. 再问了一个关于：如果前台数据无法传输到后台，该如何调试。

  3. 再者问我简历上项目经验之谈。我模模糊糊答了一些，却始终没有说清，老实说，原因有二：

    其一在于我的脑子是真的忘了。其二在于我对技术的不熟练。无法达到侃侃而谈的境地。

    心得：

    自己的不足虽然自己清楚，但这次表现的更为明显。我清楚的认识到书本上的知识还是有所用处的，不能完全抛之脑后，比如网络知识。。。然后很多以前犯的错误，即使解决了，也要及时总结和归纳分析。然后对自己做的事情要有责任感。既然自己做过的项目就应该使其发挥优势，而不是表面夸夸其谈。最后关于性格，最关键的缺点在于自信和独立。未来的路，我想我会努力变得独立，通过充实内在使自己变得自信。
